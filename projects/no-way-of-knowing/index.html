The mechanism, fully disclosed

The destructive switch works by removing the current face's DOM node entirely (Node.remove()) — never by hiding it with display, opacity, or the hidden attribute — holding a genuine empty field with zero .face nodes in the stage, and only then building the other face from scratch with document.createElement and inserting it. Returning costs the same, in reverse. The two faces are never both in the DOM. If your system prefers reduced motion, the empty field is held for less time, but it is still held, and the DOM is still destroyed and rebuilt — the destruction is the argument, not the animation.

This console does not assert that the machine-targeting program named any specific target, in either instance. Instance 1's "no way of knowing" is the state's own sentence, preserved as spoken — this studio does not resolve it. Instance 2's harm face ends on an open, unanswered question because that is the true, current state of the public record, not because this studio withheld an answer it has.
